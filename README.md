## Project Overview

This project is a structured audit of the Git version control system from both a technical and philosophical perspective. It explores how Git operates within a Linux environment, how its open-source license shapes its usage, and how it compares to proprietary alternatives.

The project also includes a set of shell scripts designed to analyze system-level aspects such as dependencies, filesystem footprint, permissions, and log activity.

---

## Objectives

- Understand the origin and philosophy behind Git
- Analyze the GPL v2 license and its implications
- Explore Git’s behavior within a Linux system
- Develop practical shell scripting skills
- Compare open-source and proprietary software models
